GitHub 專案導覽設定與部署指南

這套導覽網頁是為 GitHub Pages 量身打造的純前端專案。您可以完全在瀏覽器網頁端完成建立與部署，不需要安裝任何本機開發軟體！

1

建立 GitHub 倉庫 (Repository)

登入您的 GitHub 帳號，點擊右上角 「+」 ➔ New repository。填寫倉庫名稱（例如：my-apps-hub），將權限設為 Public (公開)，並勾選 Add a README file，點擊建立。

2

建立並貼上主網頁 index.html

在倉庫首頁，點擊 Add file ➔ Create new file。檔名填寫 index.html。將目前這個總覽網頁的整份原始碼完整複製並貼上，然後點擊右上方的 Commit changes... 存檔。

3

啟用 GitHub Pages 網頁功能

進入專案倉庫上方的 ⚙️ Settings (設定) ➔ 點擊左側選單的 Pages。 在 Branch (分支) 選項中，將來源設定為 main (或 master)，資料夾設定為 / (root)，點擊右側的 Save。

4

上傳您的小程式 ＆ 設定傳送門

如果是靜態小程式，在檔名輸入路徑：apps/sudoku/index.html 並貼上程式。 如果是不支援在 GitHub 執行的 React/AI 後端網頁，請將它們部署到外部（如 Vercel），並開啟本站管理模式，新增時選擇「外部傳送門」，填入其網址即可連結！

本專案整合了全新的「免程式碼後台視覺化編輯器」！您不需要直接編輯複雜代碼，即可隨時自訂與匯出：

💡 視覺化極速設定三步驟：

  1. 點擊右上角的 ⚙️ 管理與設定模式 按鈕進入後台。
  2. 直接在畫面上點擊各處的 ➕ 新增 或卡片上的 ✏️ 編輯 按鈕來變更設定。
  3. 修改完畢後，點擊頂端橫幅或最下方的 💾 儲存並匯出最新 HTML 程式碼。
