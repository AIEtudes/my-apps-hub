GitHub 專案導覽設定與部署指南

這套導覽網頁是為 GitHub Pages 量身打造的純前端專案。您可以完全在瀏覽器網頁端完成建立與部署，不需要安裝任何本機開發軟體！

1

建立 GitHub 倉庫 (Repository)

登入您的 GitHub 帳號，點擊右上角 「+」 ➔ New repository。填寫倉庫名稱（例如：my-apps-hub），將權限設為 Public (公開)，並勾選 Add a README file，點擊建立。

2

建立並貼上主網頁 index.html

在倉庫首頁，點擊 Add file ➔ Create new file。檔名填寫 index.html。將目前這個總覽網頁的整份原始碼完整複製並貼上，然後點擊右上方的 Commit changes... 存檔。

3

啟用 GitHub Pages 網頁功能

進入專案倉庫上方的 ⚙️ Settings (設定) ➔ 點擊左側選單的 Pages。 在 Branch (分支) 選項中，將來源設定為 main (或 master)，資料夾設定為 / (root)，點擊右側的 Save。

💡 儲存後稍等 1-2 分鐘，重新整理 Pages 設定頁，最上方就會出現您的專屬網址！(例如: https://yourname.github.io/my-apps-hub/)
4

上傳您的小程式

點擊 Add file ➔ Create new file。 在檔名輸入框輸入路徑：apps/sudoku/index.html （當您輸入斜線時 GitHub 會自動建立資料夾）。將您寫好的數獨小程式貼入並 Commit 存檔。

🚫 常見小白疑難排解 (FAQ)

Q：為什麼我更新了程式，網頁卻還是顯示舊的？
A：這通常是瀏覽器快取。請按下鍵盤 Ctrl+F5 (Mac 使用 Cmd+Shift+R) 強制重新整理，或使用無痕模式開啟。

Q：開啟小程式時出現 404 找不到檔案？
A：請確認您在下一個分頁設定的 url 與您在 GitHub 上實際儲存的資料夾/檔名大小寫完全一致。記得使用相對路徑，如：apps/sudoku/index.html。
